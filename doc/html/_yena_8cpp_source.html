122//!
123//! 3D 프로그래밍에서 장치(Device)는 시스템에 설치된 그래픽 하드웨어(어뎁터 Adaptor) 를
124//! 의미하며 넓은 의미에서 GPU 자체로 생각할 수 있습니다.
125//! 일반적인 장치들의 중요 기능(Features)은 다음과 같습니다.
126//! - 렌더링 장치(GPU), 메모리(VRAM) 운용.
127//! - 그래픽스 파이프라인 (Pipeline) 운용.
128//! - 렌더링 자원(Resource) 생성 및 관리 : VB, IB, RT, DSB 등.
129//! - 렌더링 상태(Render State) 조절.
130//! - 스왑체인(SwapChain) 운용 : 다중 버퍼링, 렌더타겟 개수, 해상도, 색상 포멧 등.
131//! - 셰이더 (Shader) 파이프라인 : 고수준 GPU 프로그래밍 HLSL, SPIR-V 지원.
132//! - 계산 파이프라인 (Compute Pipeline) 운용.
133//! - 영상 미디어 : 비디오 스트리밍 인코딩/디코딩.
134//!
135//! DirectX, OpenGL, Vulkan 같은 전문 저수준 그래픽스 API 들은 장치 운용을 위한
136//! 전용 객체(인터페이스)를 제공하며 사용자는 이를 통해 필요한 자원 생성, 작업 명령 및
137//! 장치 상태를 조작 할 수 있습니다.
138//! Yena 는 DX9 (Direct3D9) 의 장치 운용 방식을 준수하겠습니다.
139//!
140//! \par Yena 와 렌더링 장치
141//! Yena 의 SW 렌더링 장치는 HW 렌더링 장치(GPU, DX) 와 최대한 동일 기능을 지원하는 것이 목표입니다.
142//! 각 세부 기능은 B3YenaDevice9 에서 구현되며 사용자 인터페이스를 위해 IYenaDevice9 가 제공됩니다.
143//! 이 인터페이스는 DX 의 IDirect3DDevice9 을 대응합니다.
144//! Yena 렌더링 장치는 렌더링 관련 중요 SW 기능을 수행하며 학과 진행에 맞추어 많은 기능들이 추가될 것입니다.
145//!
146//! <DX> DX9 3D 렌더링 장치(Device)를 생성하려면 우선 Direct3D9 객체가 필요합니다.
147//! 이 Direct3D9 는 시스템이 보유한 물리적 장치(GPU, Adaptor) 의 지원 성능(Features)을 점검하고 렌더링 장치(Direct3DDevice9) 을 생성합니다.

149//! <Yena> IYena 와 IYenaDevice9 은 각각 IDirect3D 와 IDirect3DDevice9 을 대응하며 가능한 동일한 인터페이스로 구성됩니다.
